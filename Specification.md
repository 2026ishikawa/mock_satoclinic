# 佐藤医院 Web予約ページ 仕様書

## 1. 概要

本機能は、既存の内科医院Webサイト内に設置するWeb予約ページである。
医院紹介・アクセス・医師紹介などの静的ページはHTML/CSS/JavaScriptで作成し、予約処理が必要なページのみSpring Bootで管理する。

主な目的は、患者がWeb上で診療予約を行い、医院側が予約内容を確認できるようにすることである。

---

## 2. 機能一覧と優先度

| 優先度 | 機能 | 内容 |
|---|---|---|
| 必須 | 予約フォーム入力 | 患者情報、診療希望日、希望時間、症状を入力する |
| 必須 | 入力チェック | 必須項目、メール形式、過去日の予約不可などを検証する |
| 必須 | 予約確認 | 入力内容を確認画面で表示する |
| 必須 | 予約登録 | 予約内容をデータベースに保存する |
| 必須 | 予約完了 | 予約番号を表示する |
| 必須 | 予約枠上限チェック | 同一日時の予約数が上限を超えないようにする |
| 高 | 管理者用予約一覧 | 医院側が予約一覧を確認できる |
| 高 | 予約詳細表示 | 医院側が予約の詳細内容を確認できる |
| 中 | 予約キャンセル | 管理者が予約ステータスをキャンセルに変更できる |
| 中 | 予約検索・絞り込み | 日付、ステータスで予約を絞り込む |
| 低 | 管理者ログイン | Spring Securityで管理画面へのアクセスを制限する |
| 低 | メール通知 | 予約完了メールを患者へ送信する |

## 3. 各画面の入出力仕様

### 3.1 予約入力画面

| 項目 | 内容 |
|---|---|
| URL | `/reservations/new` |
| HTTPメソッド | GET |
| 画面目的 | 患者が予約情報を入力する |
| 入力項目 | 氏名、フリガナ、生年月日、電話番号、メールアドレス、初診/再診、予約日、予約時間、症状、個人情報同意 |
| 出力内容 | 予約フォーム |
| 遷移先 | 入力内容に問題がなければ `/reservations/confirm` |

### 3.2 予約確認画面

| 項目 | 内容 |
|---|---|
| URL | `/reservations/confirm` |
| HTTPメソッド | POST |
| 画面目的 | 入力内容を確認する |
| 入力項目 | 予約入力画面で入力された内容 |
| 出力内容 | 入力内容の確認表示 |
| バリデーション | 必須項目、形式、過去日、予約枠上限 |
| 遷移先 | 登録する場合 `/reservations`、修正する場合 `/reservations/new` |

### 3.3 予約完了画面

| 項目 | 内容 |
|---|---|
| URL | `/reservations/complete/{reservationCode}` |
| HTTPメソッド | GET |
| 画面目的 | 予約完了を通知する |
| 入力項目 | 予約番号 |
| 出力内容 | 予約完了メッセージ、予約番号、予約日時 |
| 遷移先 | トップページまたは予約入力画面 |

### 3.4 管理者用予約一覧画面

| 項目 | 内容 |
|---|---|
| URL | `/admin/reservations` |
| HTTPメソッド | GET |
| 画面目的 | 医院側が予約一覧を確認する |
| 入力項目 | 検索条件：予約日、ステータス |
| 出力内容 | 予約番号、氏名、予約日時、初診/再診、ステータス |
| 遷移先 | 予約詳細画面 `/admin/reservations/{id}` |

### 3.5 管理者用予約詳細画面

| 項目 | 内容 |
|---|---|
| URL | `/admin/reservations/{id}` |
| HTTPメソッド | GET |
| 画面目的 | 予約内容の詳細を確認する |
| 入力項目 | 予約ID |
| 出力内容 | 患者情報、予約日時、症状、ステータス |
| 操作 | キャンセル処理 |
| 遷移先 | 予約一覧画面 |

## 4. REST APIエンドポイント一覧

| HTTPメソッド | エンドポイント | 用途 |
|---|---|---|
| GET | `/api/reservation-slots` | 指定日の予約可能時間帯を取得する |
| POST | `/api/reservations/validate` | 予約内容の入力チェックを行う |
| POST | `/api/reservations` | 予約を登録する |
| GET | `/api/reservations/{reservationCode}` | 予約完了後の予約情報を取得する |
| GET | `/api/admin/reservations` | 管理者用の予約一覧を取得する |
| GET | `/api/admin/reservations/{id}` | 管理者用の予約詳細を取得する |
| PATCH | `/api/admin/reservations/{id}/cancel` | 予約をキャンセルする |

※ 画面遷移はSpring Boot + Thymeleafで実装し、必要に応じてJavaScriptから上記APIを利用する。

## 5. リクエスト/レスポンス JSON定義

### 5.1 予約可能時間帯取得

GET `/api/reservation-slots?date=2026-06-01`

レスポンス例：

```json
{
  "date": "2026-06-01",
  "slots": [
    {
      "time": "09:00",
      "available": true,
      "remaining": 2
    },
    {
      "time": "09:30",
      "available": false,
      "remaining": 0
    }
  ]
}
